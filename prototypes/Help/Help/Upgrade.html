Upgrade Software on a Managed Element
  1. Click the Upgrade tab to open the Upgrade view.

  2. Enter Upgrade Job name.

    If you do not enter a job name, a system generated name will be created automatically.

  3. Select an upgrade option:

    Prepare upgrade (SW download and verify) Verifies the ability to activate the current upgrade package and prepares the managed element for the activation of the current upgrade package. Various checks are performed on the managed element, the upgrade package is partly or fully downloaded to the managed element. Preparation does not decrease capacity on the managed element and the software version does not change.

    Activate Upgrade (Verify, activate and commit) The activation of an upgrade package means that it is taken into operation. This is the most critical part of the upgrade. During activation the capacity of the managed element decreases temporarily, that is, node performance is affected, and after activation, the software version normally changes.

    Prepare and activate upgrade
    This operation confirms the upgrade.

  4. Select the managed elements you wish to upgrade the software on.

  5. Select an upgrade package.
    If no upgrade packages are listed it is because none have been imported.

  6. Click Start.
    A confirmation dialog is displayed.

  7. Click:

    Yes to confirm.
    Once confirmed, the Jobs page is displayed and the upgrade begins immediately.

    No to cancel.

Why Prepare and Activate are separate activities

It is important to be able to prepare and activate upgrades separately because activation can impact node performance. There may not be enough time to do both activities in off-peak hours so you can prepare the upgrade while there is normal load on the nodes and activate when load is low, for example, in the early morning.
